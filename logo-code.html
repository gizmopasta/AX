BXD AI Native · 자산 코드화

로고 하나를
어떻게 코드로 만드나

파일을 공유하는 게 아니라, 링크 하나가 항상 최신을 가리키게 만드는 것. 로고 하나를 예시로 전체 과정을 풀어본다.

Overview
한 줄 요약
어디까지, 무엇을, 어떻게, 어디에.
어디까지
위키에 가이드된 범위 내에서
BXD 브랜드 가이드에 명시된 로고·심볼·아이콘 자산. 전체를 한번에 하지 않고 유형별로 하나씩 기준을 잡으며 진행.
무엇을
SVG 파일을
AI·EPS·PNG 등 혼재된 파일 중 SVG로 변환 가능한 것부터. SVG는 코드로 읽히고, 어떤 크기에서도 깨지지 않으며, AI 툴이 직접 참조할 수 있다.
어떻게
코드 변환하고,
디자인 툴에서 SVG로 내보내기 → 불필요한 코드 정리 → 용도별 버전(화이트/블랙/컬러) 분리 → 파일명 규칙 통일. 처음엔 사람이 하고, AI와 협업으로 전환하며 속도를 줄인다.
어디에
변환된 코드를 → GitHub 업로드 → 링크 지정한다
파일이 GitHub 레포에 올라가면 고정 URL이 생긴다. 이 URL이 곧 배포 링크. 이후 파일을 수정해서 다시 올리면 URL은 그대로고 내용만 바뀐다.
Process
단계별 작업
로고 하나를 기준으로, 처음부터 링크 배포까지.
1
Figma에서 SVG로 내보내기
로고 프레임 선택 → Export → SVG 선택. 이때 "Include id" 옵션 끄고, 불필요한 그룹 레이어 정리. 파일명은 규칙을 따른다.
# 파일명 규칙
ncloud-logo-color.svg ← 기본 컬러
ncloud-logo-white.svg ← 흰 배경용
ncloud-logo-dark.svg ← 다크 배경용
ncloud-symbol-color.svg ← 심볼만
2
SVG 코드 확인 및 정리
SVG 파일을 텍스트 에디터로 열어 불필요한 메타데이터 제거. width/height를 viewBox 기준으로 정리. 컬러값을 토큰과 맞게 확인.
<!-- 정리 전 -->
<svg width="200" height="48" xmlns="..." data-figma-id="...">

<!-- 정리 후 -->
<svg viewBox="0 0 200 48" xmlns="http://www.w3.org/2000/svg">
3
GitHub 레포에 업로드
BXD 전용 레포(예: gizmopasta/AX)의 assets/logo/ 폴더에 파일을 올린다. 폴더 구조가 URL 구조가 된다.
# 레포 폴더 구조
AX/
  assets/
    logo/
      ncloud-logo-color.svg
      ncloud-logo-white.svg
      ncloud-symbol-color.svg
4
링크 확인 — 이게 배포 URL
GitHub Pages가 켜져 있으면, 레포에 올라간 파일은 아래 URL로 바로 접근할 수 있다. 이 URL을 가이드 문서와 아카이빙 사이트에 등록해두면 끝.
https://gizmopasta.github.io/AX/assets/logo/ncloud-logo-color.svg

# 아이콘도 동일한 방식
https://gizmopasta.github.io/AX/assets/logo/ncloud-symbol-color.svg
5
사용 방법 — 링크를 그대로 쓴다
받는 사람은 파일을 다운로드할 필요가 없다. 링크를 Figma, Notion, 문서, 코드 어디에서든 그대로 붙여넣으면 항상 최신 버전이 로드된다.
<!-- 웹/HTML에서 -->
<img src="https://gizmopasta.github.io/AX/assets/logo/ncloud-logo-color.svg">

# Notion, 슬랙, 문서에서
링크 붙여넣기 → 미리보기로 바로 표시

Update Flow
업데이트할 때 달라지는 것
로고가 수정됐을 때 — 기존 방식과 링크 배포 방식의 차이.
지금 방식
① BXD에서 로고 파일 수정
② 새 파일 이름 바꿔서 저장
ncloud-logo-v3-final2.ai
③ 슬랙·메일로 공유 요청
④ 받는 사람이 각자 다운로드
⑤ 기존 파일 삭제하고 교체
⑥ 못 받은 팀은 구버전 계속 사용
→
링크 배포 방식
① BXD에서 로고 파일 수정
② 같은 파일명으로 GitHub에 덮어쓰기
ncloud-logo-color.svg (그대로)
③ 끝.
링크는 그대로, 내용만 바뀜
공유 안 해도 모두 최신 버전 자동 사용
구버전 사용 사고 없음
Step 1
Figma에서 수정
로고 파일을 업데이트하고 SVG로 다시 내보내기. 파일명은 동일하게 유지.
Figma
Step 2
GitHub에 덮어쓰기
같은 경로에 같은 이름으로 파일을 다시 올린다. GitHub이 버전 히스토리를 자동으로 기록.
GitHub
Step 3
링크는 그대로
배포 URL은 변하지 않는다. 이미 링크를 등록해둔 문서·사이트·코드 모두 자동으로 최신 버전을 불러온다.
자동 반영
Step 4
공유 없이 완료
"새 파일 공유드립니다" 없이 끝. 받는 사람은 쓰던 링크 그대로 쓰면 된다. BXD가 소스를 가지고 있고 배포까지 컨트롤.
BXD 컨트롤
Result
코드화 이후 달라지는 것
파일 배포 방식이 바뀌면 업무 흐름 전체가 바뀐다.
변화 1
파일 공유 요청이
사라진다
"최신 로고 파일 받을 수 있나요?" — 이 요청이 없어진다. 링크가 항상 최신이니까. BXD로 들어오는 반복 요청 감소.
변화 2
구버전 사용 사고가
없어진다
누군가 예전 파일을 갖고 있어서 구버전 로고가 결과물에 들어가는 일이 없어진다. 소스가 하나이고, 링크가 하나다.
변화 3
AI 툴이 로고를
직접 참조한다
AI에게 로고 URL을 주면 파일 없이도 최신 로고를 불러와서 쓸 수 있다. Foundation 파일에 URL을 등록해두면 모든 AI 작업에 자동 적용.
결국 이게 코드화의 의미. 파일을 관리하는 게 아니라 소스를 가지고 있는 것. BXD가 소스를 갖고 링크를 배포하면 — 업데이트도, 배포도, 버전 관리도 BXD가 컨트롤한다. 받는 사람은 그냥 링크를 쓰면 된다.
