SVG 정리 & GitHub URL

Figma에서 내보낸 SVG를 정리하는 방법, 그리고 GitHub 폴더 구조가 그대로 URL이 되는 원리.

1
SVG 파일 정리 — Before / After
Figma에서 내보내면 불필요한 코드가 많이 붙어 나온다
Figma 내보내기 직후 — 정리 전
<?xml version="1.0" encoding="UTF-8"?>
<svg
  width="200"
  height="48"
  viewBox="0 0 200 48"
  fill="none"
  xmlns:xlink="http://..."
  data-figma-id="123:456"
  data-figma-name="Logo/Color"
  xmlns="http://www.w3.org/2000/svg"
>
  <g id="Frame_1" clip-path="url(#clip0)">
    <path
      fill="#03C75A"
      d="M12 0L24 12..."
    />
  </g>
  <defs><clipPath id="clip0">...</clipPath></defs>
</svg>
정리 후 — AI가 읽기 좋은 형태
<svg
  viewBox="0 0 200 48"
  xmlns="http://www.w3.org/2000/svg"
>
  <path
    fill="#03C75A" ← 토큰값 확인
    d="M12 0L24 12..."
  />
</svg>






width / height 제거, viewBox만 남긴다
width·height가 고정되어 있으면 크기를 CSS로 조절할 수 없다. viewBox만 있으면 어떤 크기에서도 비율을 유지하며 렌더링된다.
data-figma-* 속성 전부 제거
Figma 내부에서 쓰는 메타데이터. 배포 파일에 필요 없고 파일 용량만 늘린다.
불필요한 <defs> <clipPath> 제거
단순 로고에 clip-path가 붙어 나오는 경우가 많다. 실제로 클리핑이 필요한 게 아니면 제거. 없애면 코드가 훨씬 단순해진다.
fill 컬러값을 BXD 토큰과 맞게 확인
Figma에서 내보낼 때 컬러값이 정확히 나오는지 확인. BXD 그린은 #03C75A. 비슷하지만 다른 값이 들어오는 경우가 있다.
