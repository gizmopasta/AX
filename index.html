BXD AI Native
전환 계획

업무 프로세스의 AI 자동화와 전환.
당장의 문제가 드러나지는 않지만, 언젠간 시작될 문제고 — 대응해야 한다.
브랜드 자산을 네이티브화하고, 일하면서 쌓는 아카이빙을 만들고, 실험을 시작한다.

왜 지금인가
AI를 직면했을 때 브랜드가
다시 원점이 되지 않으려면 지금 시작해야 한다
지금 방식으로도 유지는 가능하다. 그래서 당장의 문제를 해결한 게 아니니 성과가 아니라고 하면 맞다.
근데 AI에게 그리라고 했을 때 — 멋지긴 한데 우리 스타일이 아니다. 우리 그린 컬러로 만들어졌나? 우리 내용을 담고 있나? 다시 원점이다.
언제까지 일 시켜놓고 계속 보고 검수할 수만은 없다. 미래를 위한 효율적인 투자이고, 이미 있었던 부채를 해결하는 일이다.
Context
브랜드가 유독 어려운 이유
프로덕트·서비스 단에서는 이미 전환을 시작해서 테스트 단계에 있다. 브랜드는 왜 늦는가.
이유 1
개인 역량으로 커버하고 짊어져왔다
브랜드 품질은 시스템이 아니라 사람이 유지해왔다. 규칙이 아니라 감각으로. 그래서 AI에게 넘겨줄 기준 자체가 명문화되어 있지 않다.
이유 2
누구 하나 먼저 건들기 어려운 영역
범위가 너무 크고 부담감이 있다. 티가 안 날 수 있다. 그래서 시도 자체가 어렵다고 느껴진다. 근데 이것 없이 자동화를 시작하는 건 다시 원점이다.
이유 3
자산이 있어도 AI가 읽을 수 없는 형태
로고 파일 있다. 가이드 문서 있다. 근데 SVG가 없어서 코드화 안 되고, CMYK만 있어서 디지털 기준이 없고, 업데이트가 안 되어 있어서 최신 기준이 불분명하다.
방향
감각을 규칙으로, 규칙을 기준으로
분야를 들여다보고, 프로세스를 하나씩 보고, AI가 자동화할 수 있는 것을 찾아낸다. 개인 역량을 규칙으로 바꾸고 기준을 만든다. 울타리를 친다 — 범위와 제한.
Current State
지금 자산의 상태
카테고리는 있다. 근데 AI가 읽을 수 있는 형태가 아니다.

로고 & 소스파일 AI·EPS·PNG 혼재. SVG 없음. 용도별 버전 기준 없음. 코드화 불가. 모든 로고·아이콘·심볼을 SVG로 통일. 용도별 버전 정리. 링크 기반 배포로 전환 — 업데이트 시 자동 반영. → SVG + CDN
키비주얼 이미지 파일로만 보관. 구조 정보 없음. 재사용·변형 불가. 가능한 것들은 SVG 또는 구조화된 형태로 아카이빙. 색상·레이아웃 정보가 데이터로 남아야 AI가 패턴을 읽을 수 있다. → SVG / JSON
색상 토큰 CMYK만 있음. 디지털 기준 부재. 서비스별 시멘틱 컬러와 미정리. 새로 구축이 아니라 취합·점검·코드화에 집중. 서비스 시멘틱 컬러는 별개. 브랜드에서 필요한 컬러 기준만 정리. → JSON Tokens
온오프라인 가이드 업데이트 안 됨. 버전 비일관. 히스토리 확인 불가. Before·During·After 구조로 표준화. 변환 안 되는 것들은 색상·레이아웃을 텍스트 개요로 작성하고 분류. → MD
마케팅·테크 콘텐츠 아카이빙 없음. 수동 수집. PPT 산발 보관. 많이 언급되는 테크 콘텐츠·잘된 사례부터 우선 분석. 채널·포맷·캠페인 단위 분류. Brand Prompt Library 소스가 된다. → MD + 태그
행사·굿즈 히스토리 전개 확인 불가. 수동 수집. 개인 역량 의존. 비주얼 기준과 템플릿을 하나의 흐름으로 묶어 정리. AI가 비주얼 방향 가이드, 템플릿 양산 보조. → MD + Figma
핵심 원칙. 자산의 양이 문제가 아니라 AI가 읽을 수 있는 형태인가가 문제다. 전부 다 하는 게 아니라 — 유형별로 1개씩 기준을 잡으며 정규화하고, 그 기준이 나머지의 템플릿이 된다.
Roadmap
3개월 실행 계획
3개 트랙이 동시에 진행된다. SVG·코드화는 사람이 먼저 시작하고, AI와 협업으로 전환하면서 속도를 줄여가는 과정 자체가 성과 증거다.
Month 1
자산 네이티브화
수집 → 분류 → Foundation

제각각인 자산을 AI가 이해할 수 있는 포맷으로 통일한다. 사람이 먼저 기준을 잡고, AI를 붙여가며 속도를 줄인다. 전체를 한번에 정리하지 않는다.

Foundation·로고·키비주얼·가이드·콘텐츠·템플릿 각 유형의 AI 활용 가능 포맷과 필수 메타데이터 합의. 이 기준표가 이후 모든 정규화의 설계도.

Month 2
아카이빙 사이트
구축 + 파일럿 시작
일하면서 쌓는 아카이빙
정규화한 자산을 navercloudbxd.figma.site에 올린다. 사이트의 역할은 보여주는 게 아니라 — 쓰는 사람이 올바르게 활용하게 돕는 것. 동시에 파일럿 2개를 시작한다.
자산 아카이브 구축 — 태그 기반
정규화된 자산을 태그 기준대로 올린다. 검색·필터 가능한 구조. 올리는 것보다 태그와 범위·제한 기준이 잘 붙어 있는 게 핵심.
범위와 제한 기준 명시
자산마다 사용 가능 범위, 변형 허용 기준, 금지 사항을 명확하게 붙인다. 승인 없이도 스스로 판단할 수 있게. 기준이 없으면 물어보는 비용이 계속 발생한다.
AI 기능 일부 구현
Foundation 기반 브랜드 Q&A, 태그 기반 자산 추천. "이 로고 이렇게 써도 되나요?"에 사이트 안에서 바로 답이 나오는 구조. 프롬프트 사용법도 함께 제공.
추적 코드 설치
재방문 비율·페이지뷰·사용자 환경·분기별 데이터 수집. 보는 데이터가 아닌 다음 의사결정에 쓸 수 있는 데이터.
파일럿 1·2 시작

Month 3
AI 협업 전환
성과 증명 + 다음 스텝
AI 협업으로 전환 — 속도를 줄인다
사람이 하던 SVG·코드화 작업을 AI와 협업으로 전환. 걸리던 시간이 얼마나 줄었는지, 같은 리소스로 얼마나 더 많은 프로젝트를 커버했는지가 직접 성과다.
자산 정규화 AI 협업 전환
Month 1에 사람이 잡은 기준을 AI에게 넘긴다. 처리 속도 비교 데이터 기록. 자산 1개 정규화에 걸리는 시간이 얼마나 줄었는지가 증거.
Brand Prompt Library 배포
사례 아카이브와 콘텐츠에서 추출한 패턴 기반 표준 프롬프트 모음. 누가 써도 BXD 기준에 맞는 결과물이 나오는 구조. 프롬프트 사용법 가이드 포함.
파일럿 결과 도출 + Before/After 기록
비주얼 시안 파일럿·템플릿 양산 파일럿 결과. 기존 방식 대비 소요 시간·수정 횟수·산출물 수 비교. 이 데이터가 다음 스텝의 근거가 된다.
BXD 팀 내부 베타 운영 + 다음 스텝 설계
직접 써보며 막히는 지점 파악. 3개월 운영 데이터를 바탕으로 다음 범위와 우선순위 설정.
Pilot Track 1
AI 비주얼 시안 생성
실제 캠페인·행사 하나를 골라 AI 시안 초안 → BXD 완성 전 과정을 기록한다. 계획 문서가 아닌 실제 결과물이 증거.
Step 1
프로젝트 선정
실제 진행 예정 캠페인·행사 중 비주얼 시안이 필요한 것 선정.
Step 2
Foundation 주입
BXD 컨텍스트 파일을 AI에 주입. 브랜드 기준이 프롬프트의 전제.
Step 3
AI 시안 초안 생성
방향·무드·레이아웃 기준을 프롬프트로 정의. BXD 기준 안에서 복수 시안 생성.
Step 4
BXD 검토·완성
초안 방향 선택 후 BXD가 완성. 제로베이스가 아닌 기준 위에서 시작.
Step 5
Before / After 기록
소요 시간·수정 횟수·디자이너 투입 시간을 기존 방식과 나란히 기록.
파일럿의 조건. 결과가 좋아야 하는 게 아니라 과정이 기록되어야 한다. 파일럿은 실험이지 쇼케이스가 아니다.
Pilot Track 2
템플릿 AI 양산
템플릿은 많을수록, 자세할수록 유효하다. MD로 구조가 정의되면 AI가 그 위에서 변형·양산할 수 있다. 사람은 기준을 만들고, AI는 반복을 처리한다.
Step 1
기존 템플릿 해부
반복 빈도 높은 템플릿 선정. 구조·변수·고정 요소를 분해해서 MD로 명문화.
Step 2
MD 스펙 작성
고정값·변수·조건부 규칙을 MD로 정의. AI가 읽고 변형할 수 있는 형태. 아카이빙 사이트의 스펙 문서가 됨.
Step 3
AI 변형·양산 테스트
MD 스펙 주입 후 변수를 바꿔가며 변형 생성. 어디까지 AI가 만들 수 있는지 한계 테스트.
Step 4
Figma Make 연동
테스트된 MD 스펙을 Figma Make 구조로 전환. 사이트에서 스펙 확인 → 바로 양산.
우선순위
1순위 행사·이벤트 템플릿
반복 빈도 가장 높음. 행사명·날짜·장소·색상만 바뀌고 구조 동일. AI 양산 효과 즉시 체감.
2순위 마케팅 콘텐츠 템플릿
채널별 포맷이 다르지만 브랜드 기준은 동일. MD로 채널 변수를 정의하면 AI가 채널별 변형 생성.
3순위 굿즈 스펙 템플릿
소재·사이즈·인쇄 방식이 변수. MD 스펙으로 정의하면 신규 굿즈 스펙 문서를 AI가 자동 생성.
BXD가 기준을 만들고 AI가 그 기준 안에서 무한히 변형할 수 있는 구조. MD 스펙이 잘 정의될수록 AI가 만들어낼 수 있는 범위가 넓어진다.
Impact
3개월이 만드는 직접 성과
프로세스 변경 자체가 커버리지 확장이고, AI로만 가능한 새로운 산출물이 추가된다.
