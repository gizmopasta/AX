전환 계획

쌓아온 브랜드 자산을 AI가 이해하고 활용할 수 있는 형태로 전환하고,
아카이빙 사이트를 통해 가이드하는 사람과 쓰는 사람 모두가

North Star
아카이브는 보는 문서가 아니라
일하면서 꺼내 쓰는 브랜드 AI 인프라다.
감각을 규칙으로 → 파운데이션 제작 → 작은 실험 → 완벽보다 범위부터.
사이트가 이미 이 방향을 가리키고 있다. 6개월 안에 그 구조를 실제로 작동하게 만든다.

For
자산을 더 잘 활용하게 돕는 구조. 태그·범위·제한 기준이 잘 붙어 있어야 쓰는 사람이 스스로 판단할 수 있다.
Before: 아웃풋 기준 및 방향 설정

만드는 사람 — BXD
케이스가 쌓여야 기준을 추출할 수 있다
  • 근거가 있어야 규칙을 만들 수 있다
  • 히스토리 없이는 가이드 업데이트 불가
  • 아카이브 = 더 나은 경험을 위한 과정
쓰는 사람 — 조직 전체
레퍼런스 찾는 시간이 줄어든다
  • 유사 케이스에서 판단 기준을 빌릴 수 있다
  • 승인 과정이 줄고 자율성이 올라간다
  • 아카이브 = 실전 판단의 참고서
Current Assets
보유 자산 × 정규화 방향
카테고리는 있지만 포맷·품질이 제각각인 상태. AI가 브랜드 자산을 이해하고 활용하려면 파일 형태부터 정리가 필요하다.
자산 지금 상태 정규화 방향 목표 포맷
Foundation
구. BXD 시스템		 웹 문서로만 존재. 사람이 읽는 형태. AI가 브랜드 기준을 바로 읽고 적용할 수 있도록 구조화. 컬러·서체·표기 원칙을 데이터 형태로 분리. 모든 자산 정규화의 기준점이 된다. HTML → MD + JSON
로고 & 소스파일 AI·EPS·PNG 등 포맷 혼재. 용도별 버전 기준 없음. 모든 로고·아이콘·심볼을 SVG로 통일. 용도(디지털·인쇄·자수 등)별로 버전 정리. AI 툴이 파일을 직접 참조해 브랜드 기준을 유지한 채 활용할 수 있는 상태로. AI·EPS·PNG → SVG
키비주얼 이미지 파일로만 보관. 구조 정보 없음. 가능한 것들은 SVG 또는 구조화된 형태로 아카이빙. 색상·레이아웃 정보가 데이터로 남아 있어야 재사용·변형이 가능하고 AI가 패턴을 읽을 수 있다. 이미지 → SVG / JSON
사례 아카이브 파일마다 형태 다름. 맥락 정보 없음. 사례마다 배경·목적·적용 기준을 함께 기록. 이게 쌓여야 AI가 유사 상황에서 올바른 기준을 제안할 수 있다. → MD + 메타데이터
마케팅·테크 콘텐츠 채널·시기별로 산발 보관. 채널·포맷·캠페인 단위로 분류 정리. AI가 브랜드 톤과 비주얼 패턴을 학습하는 소스 데이터가 된다. → MD + 분류 태그
행사 가이드
+ 템플릿		 행사별 문서 따로, 버전 기준 없음. 비주얼 기준과 템플릿을 하나의 흐름으로 묶어 정리. AI가 비주얼 방향을 가이드하고 템플릿 반복 제작을 보조하는 구조로. (AI가 템플릿을 전부 만드는 건 다음 단계) → MD + Figma 연동
Roadmap
3개월 실행 계획
3개 트랙이 동시에 진행된다. SVG·코드화 작업은 사람이 먼저 시작하고, AI와 협업으로 전환하면서 속도를 줄여가는 과정 자체가 성과 증거가 된다.
Phase 1
Month 1
자산 정규화 + 사이트 IA
감각을 규칙으로 — 자산을 AI가 읽을 수 있게
SVG·코드화는 사람이 먼저 시작한다. 기준을 잡으면서 AI를 붙여가고, 속도가 얼마나 줄어드는지를 기록한다. 아카이빙 사이트 IA 설계도 동시에 진행해서 Month 2 구축으로 바로 이어진다.
자산 유형별 포맷 기준 정의
사례·로고·가이드·콘텐츠·템플릿 각 유형의 AI 활용 가능 포맷, 필수 메타데이터 항목, 버전 관리 기준 합의. 이 기준표가 Phase 2 아카이빙 설계의 근거.
Foundation — AI 브랜드 컨텍스트 파일로 고도화
navercloud-bxd-system.html을 Foundation으로 재분류. AI 브랜드 컨텍스트 주입 파일로 재구조화. Design Tokens JSON 공식 배포 버전 완성. MD 기반 문서도 병행 작성.
Code for AI
지금은 가설을 세우고 테스트하기 위한 md
AI를 위한 시각 언어. 수집 → 분류(패턴·근거 탐색) → Foundation → 테스트 순서로.
Foundation
어떤 단계에서 쓸지 정하고 세분화
Before(설정) · During(체크) · After(자산화) 구조로 Foundation 항목 배치.
자산 유형별 파일럿 정규화 (각 1개)
전체를 한번에 정리하려 하지 않는다. 유형별로 1개씩 기준에 맞춰 정규화하고, 그 과정에서 기준의 빈틈을 보완. 나머지 자산 정리의 실제 템플릿이 됨.
아카이빙 사이트 IA 설계
Standard → AI → For 구조 위에서, 어떤 자산이 어느 섹션에 어떤 형태로 들어가는지 정보 구조 설계. Figma Site Phase 2 구축의 설계도.
Phase 2
Month 2
아카이빙 사이트 구축 + 비주얼 AI 파일럿
파운데이션 제작 — 자산을 잘 활용하게 돕는 사이트
Month 1에서 정규화한 자산을 사이트에 올리면서 동시에, 실제 캠페인·행사 하나를 골라 AI 비주얼 시안 생성 파일럿을 시작한다. 태그 구조와 범위·제한 기준이 사이트의 핵심이고, 파일럿이 Month 3 성과 증거가 된다.
태그 체계 설계 — 찾고 쓰기 위한 구조
자산마다 용도·채널·맥락 태그를 붙인다. 태그가 잘 설계되어 있어야 검색이 되고, AI도 맥락에 맞는 자산을 제안할 수 있다. 태그 기준 자체가 브랜드 활용 기준이 된다.
범위와 제한 기준 명시 — 어디까지 써도 되는지
자산마다 사용 가능 범위, 변형 허용 기준, 금지 사항을 명확하게 붙인다. 승인 없이도 스스로 판단할 수 있게. 기준이 없으면 물어보는 비용이 계속 발생한다.
자산 아카이브 구축 — Standard 섹션
Foundation·사례·로고·키비주얼·콘텐츠를 태그 기준대로 올린다. 검색·필터가 작동하는 구조. 올리는 것보다 태그와 기준이 잘 붙어 있는 게 핵심.
AI 기능 일부 구현 — 기준을 즉답으로
Foundation 기반 브랜드 Q&A, 태그 기반 자산 추천. "이 로고 이렇게 써도 되나요?" 같은 질문에 사이트 안에서 바로 답이 나오는 구조. 범위와 제한 기준이 AI의 답변 근거가 된다.
비주얼 AI 가이드 & 템플릿 섹션
행사 가이드와 템플릿을 하나의 흐름으로 묶어 구성. AI가 비주얼 방향을 가이드하고 템플릿 반복 제작을 보조. Figma Make 연동으로 양산 효율화. (AI가 템플릿을 전부 만드는 건 다음 단계)
BXD 팀 내부 베타 운영
직접 써보며 태그와 기준이 실제로 작동하는지 확인. 막히는 지점을 Phase 3 전에 보완.
Phase 3
Month 3
AI 협업 전환 + 성과 증명
완벽보다 범위부터 — AI가 정리·분류하는 구조
사람이 하던 SVG·코드화 작업을 AI와 협업으로 전환한다. 걸리던 시간이 얼마나 줄었는지, 같은 리소스로 얼마나 더 많은 프로젝트를 커버했는지가 이 단계의 직접 성과다. 비주얼 AI 파일럿 결과도 이 시점에 나온다.
자산 업로드 → 자동 메타데이터 생성
파일 업로드 시 AI가 유형·카테고리·태그·사용 맥락 자동 추출. Phase 1 포맷 기준이 AI 분류의 학습 데이터가 됨.
Brand Prompt Library 공식 배포

전사 오픈 — 브랜드 AI 인프라 v1.0
내부 검증 완료 후 조직 전체에 공개. 6개월 운영 결과를 바탕으로 BXD의 AI 인프라 역할·범위 공식 정의. 규칙·우선순위·근거 세 가지를 갖춘 v1.
Next — 검수·검열 AI. 산출물이 BXD 기준에 맞는지 AI가 자동으로 검수하는 기능. 컬러·서체·간격·표기 원칙을 Foundation 기준으로 대조하고 이슈를 리포트. v1 인프라가 안정화된 이후 단계로, 6개월 계획의 범위 밖이지만 방향은 지금부터 설계에 반영해둔다.

Impact
3개월이 만드는 직접 성과
프로세스 변경 자체가 커버리지 확장이고, AI로만 가능한 새로운 산출물이 추가된다.
성과 1 — 프로세스
같은 인원으로
더 많은 범위를 커버
지금은 BXD가 직접 관여해야 품질이 유지된다. AI 전환 후엔 BXD 없이도 브랜드 기준이 적용된다.
SVG 변환·코드화를 디자이너가 하나씩 수작업. 자산 하나에 걸리는 시간이 일정하지 않음
→
사람이 기준 잡고 시작 → AI가 반복 처리 인수. 같은 시간에 처리 가능한 자산 수가 늘어난다
타팀 브랜드 적용 요청 → BXD 검토 → 피드백 → 수정. 사이클마다 BXD 리소스 소모
→
태그·범위·제한 기준이 사이트에 있으니 타팀이 스스로 판단. BXD 직접 개입 없이 커버 가능한 프로젝트·채널 수 증가
성과 2 — 새로운 산출물
AI 비주얼 시안 생성
파일럿
이전에는 불가능했던 것. BXD 기준을 학습한 AI가 시안 초안을 직접 생성한다.
캠페인·행사 비주얼은 디자이너가 제로베이스에서 시작. 레퍼런스 수집 → 컨셉 → 시안까지 시간 소요
→
Foundation 기반 AI가 BXD 기준 시안 초안 생성. 디자이너는 제로베이스가 아닌 기준 위에서 시작. 컨셉 완성도와 속도 동시 향상
Month 2에 시작 → Month 3에 결과 도출
실제 프로젝트 하나를 골라 AI 시안 생성 → BXD 검토·완성 전 과정을 기록. 기존 방식 대비 시안 도출까지 걸린 시간, 수정 횟수, 산출물 완성도가 증거가 된다.
성과 3 — 자산 가치
쌓아온 자산이

태그 기반 검색으로 유사 사례를 즉시 찾고, 기준을 확인해서 바로 적용. 자산 활용 빈도가 올라가고, BXD가 만든 것들이 실제 업무에서 참조되기 시작
3개월 후 BXD가 말할 수 있어야 하는 것. "같은 인원으로 커버하는 프로젝트가 늘었다. AI가 시안을 만들기 시작했다. 자산이 실제로 쓰이고 있다." 이 세 가지가 이 전환의 직접 성과다.
Pilot Track
AI 비주얼 시안 파일럿
3개월 계획과 동시에 진행하는 별도 트랙. 실제 프로젝트 하나를 처음부터 끝까지 AI와 함께 만들어 성과의 증거로 남긴다.
What & Why
실제 캠페인·행사 하나를 골라
AI 시안 초안 → BXD 완성 전 과정을 기록한다
계획 문서가 아닌 실제 결과물이 증거가 된다. 기존 방식과 비교해서 시간·수정 횟수·완성도를 나란히 놓을 수 있어야 한다.
Step 1
프로젝트 선정
실제 진행 예정인 캠페인 또는 행사 하나를 선정. 비주얼 시안이 필요한 것이어야 함.
Step 2
Foundation 주입
BXD 컨텍스트 파일(컬러·서체·비주얼 원칙)을 AI에 주입. 브랜드 기준이 프롬프트의 전제가 된다.
Step 3
AI 시안 초안 생성
방향·무드·레이아웃 기준을 프롬프트로 정의. AI가 BXD 기준 안에서 복수의 시안 초안을 생성.
Step 4
BXD 검토·완성
초안 중 방향을 선택하고 BXD가 완성도 높인다. 제로베이스가 아닌 기준 위에서 시작하는 차이를 기록.
Step 5
Before / After 기록
기존 방식 대비 소요 시간, 수정 횟수, 초안 품질을 나란히 기록. 이게 증거가 된다.
