BXD AI Native · 컬러

Color를
AI 네이티브화한다

새로 구축하는 게 아니다. 이미 있는 컬러를 취합하고, 점검하고, AI가 읽을 수 있는 형태로 코드화하는 것. 서비스 시멘틱 컬러와 분리해서 브랜드에서 필요한 부분에 집중한다.

1
범위 — 무엇을 하고, 무엇을 안 하는가
새로 만드는 게 아니라 정리하는 것
이번에 한다
브랜드 컬러 취합·점검·코드화
온오프라인 기준 BXD 브랜드 컬러 전체 취합
CMYK → RGB → HEX 값 점검 및 통일
디지털 기준 컬러값 JSON 토큰화
용도별 컬러 기준 명문화 (디지털/인쇄/자수 등)
AI 툴에 주입할 수 있는 형태로 구조화
이번에 안 한다
서비스 시멘틱 컬러 신규 구축
각 서비스·프로덕트의 시멘틱 컬러 시스템은 별개
이미 서비스별로 개별 시멘틱 컬러 보유 중
프로덕트팀에서 필요한 내용도 아님
새 시멘틱 컬러 시스템 신규 구축은 이번 범위 밖
브랜드에서 필요한 부분만 — 모으고 코드화
핵심. 서비스랑 브랜드는 별개다. 서비스는 이미 각자 시멘틱 컬러를 가지고 있다. 이번 기회에 가지고 있는 것들을 취합하고 점검하는 정도로 하고, 코드화하는 것에 집중한다.
2
지금 컬러가 AI에게 안 먹히는 이유
AI가 읽을 수 없는 형태라서
지금 상태 — AI가 읽지 못한다
"BXD 그린을 써줘"
→ AI: 어떤 그린? 어떤 값?
#00FF00? #4CAF50? 모름

BXD 가이드 PDF 첨부
→ AI: CMYK 70/0/100/0 읽음
근데 디지털 HEX값 없음
변환값 추정 → 틀릴 수 있음

매번 "#03C75A 써줘" 설명
→ 대화마다 반복. 체계 없음
코드화 이후 — AI가 바로 읽는다
"BXD Foundation 파일 첨부"
→ AI: 토큰 파일 읽음
brand-green = #03C75A 인식

"BXD 그린 계열로 만들어줘"
→ AI: brand-green #03C75A
brand-green-dark #02A84B
자동 적용

매번 설명 불필요
→ Foundation 파일 한 번 주입으로 해결
CMYK는 인쇄 기준 — 디지털에서 AI가 쓰려면 HEX가 필요하다
지금 가이드에 CMYK만 있는 경우가 많다. AI 툴은 디지털 환경이라 HEX·RGB 기준이 없으면 정확한 컬러를 쓸 수 없다. CMYK에서 변환하면 미묘하게 달라질 수 있어서 BXD가 직접 디지털 기준값을 정의해야 한다.
"BXD 그린" 같은 이름만으론 AI가 모른다
말로 설명해도 AI는 정확한 값을 모른다. 토큰 파일 형태로 이름과 값을 매핑해줘야 한다. brand-green: #03C75A 이런 식으로.
3
BXD 브랜드 컬러 체계 정리
취합·점검 후 이 구조로 정리한다
Brand Green
#03C75A
메인 브랜드 컬러
Green Dark
#02A84B
강조·hover 상태
Brand Black
#000000
텍스트·배경
Brand White
#FFFFFF
배경·역전 텍스트
Background
#F0F2F5
서브 배경
실제 값은 취합 후 확인·수정 필요. 위 값은 예시. 온오프라인 가이드에 흩어진 컬러값을 모아 BXD가 직접 디지털 기준값을 확정해야 한다.
// bxd-color-tokens.json
// AI 툴에 첨부하면 이 이름으로 컬러를 인식한다

{
  "brand": {
    "green": {
      "default": "#03C75A", ← 메인 브랜드 그린
      "dark": "#02A84B", ← 강조·hover
      "light": "#E8F9F0" ← 배경·틴트
    },
    "black": "#000000",
    "white": "#FFFFFF",
    "bg": "#F0F2F5"
  },
  "print": {
    "green-cmyk": "C70 M0 Y100 K0", ← 인쇄용 별도 보관
    "green-pantone": "Pantone 7480C"
  }
}

# Foundation MD 파일에 링크로 등록

# AI 프롬프트에서 URL 직접 참조
"다음 URL의 컬러 기준을 따라줘:
 https://gizmopasta.github.io/AX/foundation/bxd-color-tokens.json"
파일을 첨부하거나 URL을 주면 AI가 직접 읽는다
Claude 같은 AI 툴은 JSON 파일을 첨부하면 내용을 바로 파싱한다. GitHub에 올린 URL을 주면 링크를 통해 최신 파일을 불러온다. 항상 최신 토큰이 적용된다.
컬러 업데이트 시 JSON 파일만 수정하면 끝
컬러값이 바뀌면 bxd-color-tokens.json 파일 하나만 고쳐서 GitHub에 올린다. URL은 그대로이고, 그 링크를 쓰는 모든 AI 작업·문서에 자동 반영된다.
5
컬러 AI 네이티브화 전체 흐름
취합 → 점검 → 코드화 → 배포 → 적용
Step 1
흩어진 컬러 취합
온오프라인 가이드, PDF, 피그마 파일에 흩어진 컬러값 전부 모은다. CMYK·Pantone·HEX 혼재 상태 그대로 일단 모두 수집.
수집
Step 2
디지털 기준값 확정
CMYK → HEX 변환 점검. 비슷하지만 다른 값들 통일. BXD가 디지털 기준 HEX값을 직접 확정한다. 추정값 금지.
점검
Step 3
JSON 토큰 파일 작성
확정된 값을 bxd-color-tokens.json으로 작성. 이름·용도·값을 구조화. 인쇄용 CMYK·Pantone도 별도 항목으로 보관.
코드화
Step 4
GitHub 업로드 & URL 배포
AX/foundation/ 폴더에 올린다. URL이 생긴다. Foundation MD 파일과 아카이빙 사이트에 링크 등록.
배포
Step 5
AI 작업에 적용
파일 첨부 또는 URL로 AI에 주입. 브랜드 프롬프트 라이브러리에 표준 사용법 등록. 누가 써도 동일한 컬러 기준 적용.
적용
컬러가 코드화되면 시작과 끝 모두에서 기준이 생긴다. AI가 만들 때 BXD 컬러 기준으로 시작하고, 검수할 때도 같은 기준으로 대조할 수 있다. 지금은 둘 다 사람이 감으로 판단하고 있다.
