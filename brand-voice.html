BXD AI Native · Foundation

Brand Voice를
Foundation에 넣는다

컬러·로고만 있는 Foundation은 AI에게 "어떻게 생겼는지"만 알려준다.
Brand Voice가 추가되면 AI가 "왜 이렇게 만드는지"를 이해하고 판단할 수 있다.

1
왜 Brand Voice가 Foundation에 있어야 하는가
기획뿐 아니라 모든 AI 작업에 영향을 준다
영향 1
카피·텍스트 생성의 기준
AI가 SNS 포스팅, 행사 소개문, 제품 설명을 쓸 때 — Brand Voice가 없으면 그냥 그럴듯한 문장을 만든다. 있으면 BXD의 언어로 만든다.
영향 2
비주얼 방향 판단의 근거
시안을 만들 때 "화려함보다 명확함"이라는 한 줄이 있으면 AI가 방향을 잡는 기준이 생긴다. 없으면 매번 설명해야 한다.
영향 3
검수 AI의 판단 기준
나중에 검수 AI를 붙이면 컬러·서체뿐 아니라 "이 카피가 우리 톤에 맞는가"도 판단할 수 있다. Brand Voice가 Foundation에 있어야 가능한 일.
Brand Voice 없을 때
"네이버클라우드 행사 소개문 써줘"
→ 일반적인 IT 행사 문투. 어디서나 볼 수 있는 문장.
"배너 시안 만들어줘"
→ 트렌디하고 화려한 디자인. BXD 기준인지 알 수 없음.
매 작업마다 "우리는 B2B야, 신뢰 중심이야" 반복 설명.
→
Brand Voice 있을 때
"Foundation 파일 참고해서 행사 소개문 써줘"
→ B2B 전문가 대상, 신뢰·기술력 기반 문투 자동 적용.
"Foundation 기준으로 배너 만들어줘"
→ 명확함 우선, 과장 없이, BXD 톤의 비주얼.
설명 없이도 BXD 언어로 작동. 누가 써도 일관성 유지.
2
Brand Voice — 설계해야 할 항목
AI가 판단 기준으로 쓸 수 있는 형태로 써야 한다
항목 01
브랜드 컨텍스트
Who we are
AI에게 우리가 누구인지 알려주는 전제. 단순 소개가 아니라 AI가 판단 기준으로 쓸 수 있는 문장으로.
우리는 어떤 회사인가 — 산업·규모·포지션
예: "국내 1위 퍼블릭 클라우드 기업. B2B 중심. 기업·공공·스타트업 대상."
우리가 해결하는 문제는 무엇인가
예: "기업의 디지털 전환과 AI 도입. 기술 복잡성을 낮추고 신뢰할 수 있는 인프라를 제공."
우리의 주 대상은 누구인가
예: "CTO, 개발자, IT 의사결정자. 기술적 판단 능력을 가진 전문가."
항목 02
브랜드 가치 & 성격
What we stand for
우리가 중요하게 여기는 것. 추상적인 가치가 아니라 AI가 실제 표현에 적용할 수 있는 구체적인 문장으로.
핵심 가치 3–5개 — 그리고 각각이 표현에서 무엇을 의미하는지
예: "신뢰 → 과장 없이 사실 기반. 수치와 사례로 말한다. '최고'보다 '검증된'."
우리 브랜드의 성격을 사람으로 표현하면
예: "믿을 수 있는 기술 전문가. 허세 없이 실력으로 말하는 사람."
항목 03
톤 스펙트럼
How we sound
우리 브랜드의 말투를 스펙트럼으로 정의한다. 어느 쪽에 가까운지 위치를 정해두면 AI가 표현 수위를 조절할 수 있다.
격식체
친근체
기술적
일반적
간결
상세
절제
감성적
위 스펙트럼은 예시 위치. BXD가 실제 위치를 토론해서 확정해야 한다. 위치를 정하는 과정 자체가 브랜드 언어를 합의하는 작업.
항목 04
Do / Don't
표현 기준
AI가 가장 직접적으로 쓰는 부분. 해도 되는 표현과 하면 안 되는 표현을 구체적인 예시로.
쓰는 표현 — 구체적 수치, 사실 기반, 기술 용어 적절히
예: "99.9% 가용성 보장" O / "압도적인 안정성" X
안 쓰는 표현 — 과장 수식어, 감탄사, 불필요한 강조
예: "최고의", "혁신적인", "놀라운" — 이런 단어들은 B2B 전문가에게 신뢰를 깎는다.
채널별 톤 조정 기준
예: 기술 블로그는 전문용어 허용 / SNS는 한 문장 더 쉽게 / 행사 소개는 약간 더 따뜻하게.
항목 05
비주얼 무드 언어
시각 표현 기준
텍스트 톤뿐 아니라 비주얼 방향도 언어로 정의한다. 시안을 만들 때 AI가 방향 기준으로 참조한다.
비주얼이 주는 첫인상 — 3개 형용사
예: "명확한, 신뢰할 수 있는, 앞서가는"
비주얼에서 피해야 할 것
예: "과도한 그라디언트, 유행 중심 디자인, 소비자 브랜드처럼 보이는 것"
참고 레퍼런스 — 우리와 결이 비슷한 브랜드
AI가 방향을 잡을 때 레퍼런스 이름만 있어도 큰 도움이 됨.
항목 06
System Prompt 형태
AI 주입용
위 항목들을 AI가 바로 읽고 적용할 수 있는 System Prompt 형태로 변환한다. Foundation 파일에 이 버전이 포함되어야 실제로 작동한다.
짧은 버전 — 프롬프트 앞에 붙이는 1–3문장
빠른 작업에서 붙여쓰는 용도. "당신은 NAVER Cloud BXD의 브랜드 전문가입니다. B2B 클라우드 기업, 신뢰·기술력 중심, 전문가 대상으로 커뮤니케이션합니다."
긴 버전 — Foundation 파일 첨부용 전체 컨텍스트
위 항목 01–05를 모두 포함한 구조화된 텍스트. 중요한 작업에 파일째 첨부.
3
Foundation 파일 안에서 어떻게 생겼는가
컬러·로고 토큰과 같은 파일 안에
// bxd-foundation.json
// 컬러·로고와 같은 파일 안에 voice 섹션으로 들어간다

{
  "brand": {
    "color": { ... 컬러 토큰 ... },
    "logo": { ... 로고 URL ... },

    "voice": {
      "context": "국내 1위 퍼블릭 클라우드 기업. B2B 중심. 기업·공공·스타트업 대상.",
      "audience": "CTO, 개발자, IT 의사결정자. 기술적 판단 능력을 가진 전문가.",
      "values": [
        "신뢰: 과장 없이 사실 기반. 수치와 사례로 말한다.",
        "기술력: 트렌드보다 검증된 기술. 실력으로 말한다.",
        "명확함: 복잡한 것을 쉽게. 화려함보다 이해하기 쉽게."
      ],
      "tone": {
        "do": ["구체적 수치", "사실 기반", "기술 용어 적절히"],
        "dont": ["최고의", "혁신적인", "놀라운", "압도적인"]
      },
      "visual-mood": ["명확한", "신뢰할 수 있는", "앞서가는"],
      "system-prompt-short": "당신은 NAVER Cloud BXD 브랜드 전문가입니다. B2B 클라우드 기업, 신뢰·기술력 중심, IT 전문가 대상으로 커뮤니케이션합니다. 과장 없이 사실 기반으로, 명확하게 표현합니다."
    }
  }
}
위 내용은 예시. 실제 값은 BXD 팀이 직접 작성해야 한다. JSON 구조만 맞추면 되고 — 내용을 채우는 작업이 곧 브랜드 보이스를 합의하는 과정이다. 이 합의 자체가 AI 작업보다 더 중요한 결과물일 수 있다.
4
어떻게 만드는가 — 작업 순서
워크숍 → 초안 → 검증 → Foundation 통합
Step 1
BXD 팀 워크숍
항목 01–05의 질문들을 팀이 함께 답한다. 합의되지 않은 부분을 드러내는 것 자체가 목적.
Step 2
초안 작성
워크숍 결과를 바탕으로 각 항목 초안 작성. AI 도구로 초안 생성 후 BXD가 다듬는 방식도 유효.
Step 3
실제 작업에 테스트
초안을 AI에 주입하고 실제 카피·시안을 만들어본다. "이게 우리 말투 맞나?"를 팀이 함께 판단.
Step 4
Foundation 통합
검증된 내용을 JSON 형태로 정리해 Foundation 파일에 통합. GitHub 배포 후 모든 AI 작업에 적용.
브랜드 보이스를 만드는 과정 자체가 성과다. AI를 위한 문서를 만들면서 팀이 "우리가 어떤 브랜드인가"를 합의하게 된다. 이 합의는 AI 작업뿐 아니라 모든 브랜드 판단의 기준이 된다.
